# Schema Reference

JSON schemas used by the lazy-skill-write toolchain.

## evals.json

Located at `<skill-dir>/evals/evals.json`. Defines test cases for evaluating skill trigger behavior and output quality.

```json
{
  "skill": {
    "name": "string — skill name from frontmatter",
    "description": "string — skill description from frontmatter"
  },
  "evals": [
    {
      "id": "string — unique identifier, e.g. 'should-trigger-basic'",
      "type": "should_trigger | should_not_trigger | boundary | verification",
      "prompt": "string — the user message to test against",
      "expected": "string — what should happen",
      "expectations": [
        "string — specific verifiable assertion"
      ],
      "input_files": [
        "string — optional, relative paths to input files for the test"
      ]
    }
  ]
}
```

**Eval types:**
- `should_trigger`: skill must activate for this prompt
- `should_not_trigger`: skill must NOT activate for this prompt
- `boundary`: edge-case inputs — test graceful handling
- `verification`: tests a specific postcondition or hard gate

## eval_results.json

Generated by `scripts/run_eval.py` at `<skill-dir>/evals/eval_results.json`.

```json
{
  "skill": "string — skill name",
  "total": "number — total eval cases",
  "passed": "number — cases that passed",
  "failed": "number — cases that failed",
  "pass_rate": "number — 0.0 to 1.0",
  "results": [
    {
      "id": "string — eval case id",
      "type": "string — eval type",
      "prompt": "string — test prompt",
      "triggered": "boolean — whether the skill triggered",
      "passed": "boolean — whether the result matches expectations"
    }
  ]
}
```

## review.json

Generated by the reviewer agent at `<skill-dir>/review.json`.

```json
{
  "skill_name": "string",
  "scores": {
    "contract": "number — 0-10",
    "execution": "number — 0-10",
    "verification": "number — 0-10",
    "feedback": "number — 0-10",
    "overall": "number — 0-10, weighted average"
  },
  "issues": [
    {
      "layer": "contract | execution | verification | feedback",
      "severity": "critical | major | minor",
      "description": "string — what's wrong",
      "suggestion": "string — how to fix it",
      "line": "number | null — line in SKILL.md"
    }
  ],
  "summary": "string — one paragraph assessment"
}
```

## SKILL.md Frontmatter

Required and optional fields in the YAML frontmatter:

```yaml
# Required
name: string          # kebab-case, max 64 chars
description: string   # max 1024 chars, third person, WHAT + WHEN
version: string       # semver, e.g. "0.1.0"
author: string        # author identifier

# Recommended
tags: string[]        # searchable tags
category: string      # e.g. dev-tools, writing, ops, data, support
platforms: string[]   # e.g. claude-code, cursor
visibility: string    # public or private

# Optional
use_cases: string[]   # concrete usage scenarios
license: string       # e.g. MIT, Apache-2.0
```
